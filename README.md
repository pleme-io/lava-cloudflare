# lava-cloudflare

Autogenerated typed `(deflava-resource …)` shapes for every Cloudflare
terraform-provider resource. Mirrors pangea-cloudflare.

## Layout

```
schema.json           # terraform-provider-cloudflare schema (source of truth)
resources/            # generated typed shapes (one .tlisp per resource)
  cloudflare_zone.tlisp
  cloudflare_dns_record.tlisp
  ...
  _index.tlisp        # catalog aggregator (deflava-provider :cloudflare)
src/lib.rs            # RESOURCES constant + on-disk integrity tests
```

## Regenerate

```bash
tofu providers schema -json > schema.json
lava-forge generate --schema schema.json --out resources
```

The `schema.json` checked in is seeded with the most-used Cloudflare
resources (zone / dns_record / r2_bucket{,_cors,_custom_domain} /
tunnel{,_config} / zone_settings_override). Re-run the generator
against a full provider schema to expand coverage.

## Consumer pattern

```rust
use lava_cloudflare::RESOURCES;
for name in RESOURCES {
    // load `resources/<name>.tlisp` via include_str!() or fs::read.
}
```
